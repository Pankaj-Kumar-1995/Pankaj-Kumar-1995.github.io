## Image Guidelines

### Naming Conventions
- Use descriptive, lowercase names with hyphens
- Examples: `project-cover.jpg`, `process-flow.png`, `skill-badge.svg`
- Avoid spaces and special characters

### Format Recommendations
- **Logos & Icons:** PNG (supports transparency)
- **Photos & Complex Graphics:** JPG (compressed)
- **Diagrams & Workflows:** PNG or SVG (vector for scalability)
- **Screenshots:** PNG (lossless compression)

### Sizing Guidelines
- Cover images: 1440x810px (widescreen 16:9)
- Logos: 256x256px (scalable vector preferred)
- Thumbnails: 300x200px
- Full-width images: 1200x800px

## Adding New Images

1. Determine which section the image belongs to
2. Name the file descriptively (e.g., `project-name-screenshot.png`)
3. Place it in the appropriate folder
4. Update the relevant component or data file path:
   - Format: `/images/[section]/[filename]`
   - Example: `/images/projects/new-project-cover.jpg`

## Maintenance

- Review unused images quarterly
- Keep file sizes optimized (compress before uploading)
- Update this README when adding new image categories
- Archive old images in a separate folder if needed
